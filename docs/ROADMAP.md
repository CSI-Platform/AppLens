# AppLens Roadmap

## Platform Preview

- Position AppLens as CSI's local control board for apps, agents, evidence, approvals, and exports.
- Keep Scanner and Tune as first-party modules.
- Use the proprietary blackboard as the module coordination layer.
- Keep the WinUI shell as the local operator surface.
- Keep Microsoft Store packaging as a distribution track.

## Current Capabilities

- Cross-platform Scanner scripts.
- Native desktop Scanner and Tune collectors.
- Readiness score and tune-plan guidance.
- Blackboard event/store primitives.
- Module manifests and module status checks.
- Dashboard read models.
- JSON, Markdown, local HTML, and script text exports.
- MSIX package smoke build.

## Next Platform Moves

- Route desktop Tune execution through the platform loop.
- Record proposal, approval, execution, and verification in the blackboard.
- Treat optional external modules as configured capabilities.
- Make module cards and pending approvals the primary shell experience.
- Define Planner records for multi-step local work.

## Future Modules

- **Fleet**: manage blackboard-backed state across approved machines.
- **RAG**: govern local and private retrieval assets.
- **MCP**: expose controlled module actions through explicit tool contracts.
- **Gov**: policy, approval, audit, and export governance.

## Packaging Track

- Replace placeholder branding with final CSI assets.
- Reserve final Partner Center app identity.
- Add hosted privacy policy and support URLs.
- Capture Store screenshots from the final UI.
- Run Windows App Certification Kit.
- Produce signed MSIX package through the Store submission flow.
